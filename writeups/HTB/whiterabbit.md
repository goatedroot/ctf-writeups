## privilege escalation — morpheus to neo (reverse engineering)

earlier we saw the command:

```bash
cd /home/neo/ && /opt/neo-password-generator/neo-password-generator | passwd
```

which means user neo has a password generated by `neo-password-generator`

using `strings neo-password-generator` we got some crucial info:

```
1. It's a C program (main.c, GCC: (GNU) 14.2.1 20240805)
2. Uses srand() with gettimeofday() for seeding random generator
3. Character set: abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789 (62 chars)
```

and from the `command_log` we had the time he created his password at: `2024-08-30 14:40:42`

after trying to use faketime to manipulate time for it, it still didnt work.

then i thought it would be a great time to learn reverse engineering so i launched up ghidra and imported the file after curling it to my machine. then after clicking yes to analyze, in the symbol tree there was function `main` and `generatepassword`, clicking on which i got decompiled code and found out: the seed only changes every millisecond, not every microsecond! which means we have to find the password out of a thousand milliseconds not a million microseconds lol

the decompiled code:

```c
gettimeofday(&tv, NULL);
seed = tv.tv_sec * 1000 + tv.tv_usec / 1000;  // tv_sec = seconds, tv_usec = microseconds
srand(seed);

for(i=0; i<20; i++) {
    password[i] = charset[rand() % 62];
}
```

after trying for a while and making deepseek generate many code it wasnt working so when i tried curling the script to generate passwords to the CTF machine and then generating passwords there, then it worked!! (cuz of different C libraries)

after getting the pass i `su`'d into neo
